## 1. 这个可视化要回答什么问题

这个项目关注一个看似简单、实际很复杂的问题：一个国家是否能够在继续变富的同时减少碳排放？

更具体地说，项目希望回答四个层次的问题：

- 哪些国家已经实现了经济增长与人均碳排放下降同时发生的“绝对脱钩”？
- 这种脱钩从什么时候开始出现，轨迹上是否能看到明确的转折点？
- 脱钩背后的机制是什么，是能源结构变化、产业结构调整、效率提升，还是把高碳生产转移到了海外？
- 不同发展阶段的国家是否应当被放在同一个标准下比较，还是需要区分成熟经济体、新兴工业化国家与高收入消费型国家？

因此，本项目并不只把国家分成“减排成功”和“减排失败”两类，而是把 1990 至 2022 年的国家轨迹放在同一套坐标系统中，观察它们如何移动、何时掉头，以及这种掉头是否真实。

## 2. 数据与处理方式

项目使用国家年度面板数据，核心时间范围为 1990 至 2022 年。原始数据文件放在 `webapp/data/raw`，构建时由 `scripts/prepare-data.mjs` 清洗并生成前端可读取的 JSON 文件。

主要数据来源包括：

- Our World in Data CO2 and Greenhouse Gas Emissions dataset：提供国家年度 CO2 排放、生产端排放、人均排放、消费端排放、GDP、人口等字段。
- Our World in Data Energy dataset：提供一次能源消费、化石能源占比、低碳能源占比、可再生能源占比、煤炭占比等能源结构字段。
- World Bank country metadata：用于补充国家代码、地区、收入组等元数据。

项目在原始字段基础上计算了若干派生指标：

- `gdpPerCapita`：人均 GDP，用于衡量经济增长。
- `co2PerCapita`：人均生产端 CO2 排放，用于衡量本土生产活动产生的碳排放。
- `consumptionCo2PerCapita`：人均消费端 CO2 排放，用于观察进口商品中隐含的碳排放。
- `carbonIntensity`：每 1,000 美元 GDP 对应的 CO2 排放吨数，用于衡量经济活动的碳效率。
- `consumptionProductionGap`：消费端与生产端排放差值，用于判断一个国家是否可能通过外包高碳生产获得“表面脱钩”。
- `decouplingStartYear`：连续出现 GDP 增长且人均 CO2 下降的起始年份。
- `turningPointYear`：排放峰值或轨迹转向发生的年份。
- `trajectoryType`：根据 GDP 与排放变化将国家划分为绝对脱钩、相对脱钩、增长伴随排放、波动/数据不足等类型。

为了避免极端值压缩大多数国家的可读空间，概览图采用坐标轴截断和边缘标记：超出核心范围的国家不会被删除，而是贴在图表边缘显示。气泡半径也设置了最大值，避免人口大国完全遮挡其他国家。

## 3. 页面结构与叙事设计

页面采用“故事叙事 + 自由探索”的双层结构。

第一层是六步 guided tour。读者向下阅读时，右侧图表会自动切换到对应步骤的国家、年份和指标：先看全球象限，再突出真正进入右下象限的绝对脱钩国家，然后比较英国与德国、美国与印度，接着切换到消费端排放，最后进入自由探索。每个关键步骤下方都会给出 evidence strip，例如煤炭占比、低碳电力占比、碳强度变化或消费端差值，避免叙事判断脱离数据证据。

第二层是数据探索。读者可以在全球散点图中缩放、悬停、点击和框选国家，再进入国家路径、机制解释与类型总结。这部分保留了分析工具的自由度，让读者在读完故事后继续验证自己的判断。

最终页面主要包含四个模块：

- 故事部分：用六步 guided tour 展示从全局判断到机制追问的完整阅读路径。
- 全球图谱：用散点图展示各国 GDP 与排放变化，并支持缩放、悬停、点击和框选；框选后会生成选择集合摘要。
- 路径与机制：用连线散点图展示国家在 GDP 与排放坐标中的年度轨迹，并补充能源结构、碳强度、生产端与消费端差异。
- 国家分型：用小多图总结不同脱钩类型的典型路径，并公开每类的分类规则。

## 4. 视觉编码与交互设计决策

概览散点图的横轴是人均 GDP 变化，纵轴是排放指标变化。向右代表经济增长，向下代表排放下降，因此右下象限就是最核心的“绝对脱钩”区域。颜色表示国家类型，气泡大小表示终点年份人口规模。这个设计可以让读者在一张图上同时看到增长、排放、人口与类型。

路径图采用连线散点图，而不是普通折线图。横轴是人均 GDP，纵轴是人均排放，线条连接一个国家从起点年份到终点年份的年度位置。如果轨迹向右上移动，说明经济增长仍伴随排放上升；如果轨迹向右下掉头，说明经济继续增长但排放开始下降。这个视觉形态比单独两条折线更直接，因为“脱钩”本质上就是二维关系的方向改变。

机制解释部分加入了生产端与消费端排放对比。这样做是为了避免把所有生产端排放下降都解释为真实减排。对于高收入国家，如果消费端排放明显高于生产端排放，说明其消费仍然依赖海外高碳生产，脱钩质量需要被重新审视。

在多国比较时，项目进一步加入了 production vs consumption dumbbell plot。它把同一国家的生产端人均排放变化和消费端人均排放变化放在一条线上，帮助读者快速判断：生产端看起来下降明显的国家，在消费端修正后是否仍然保持同样幅度的改善。

交互上，项目重点实现了几类操作：

- 滚动驱动图表切换：guided tour 步骤进入视口时，右侧图表自动更新。
- 章节点击跳转：用户也可以直接点击章节标题切换到对应图表。
- 悬停高亮：在故事图和探索图中，鼠标靠近国家轨迹或点位时会高亮并显示关键信息。
- 语义缩放：散点图可以放大，但点的半径不会跟着无限变大，避免放大后仍然拥挤。
- 框选与取消选择：用户可以刷选一组国家，也可以再次点击已选国家取消选择。刷选后，信息面板会计算集合特征，包括主要地区、高收入占比、平均 GDP 变化、平均排放变化、平均可再生能源变化和消费端高于生产端的国家数。
- 聚光灯模式：选中国家时，其他国家弱化，帮助读者把注意力集中到当前对象。

## 5. 被考虑但没有采用的方案

最初方案更接近传统 Dashboard：顶部放置几个按钮，点击后切换图表状态。这个方案实现简单，但阅读体验较割裂，用户需要先理解按钮含义，再主动决定看什么。最终改为滚动叙事，是因为项目的核心目标不是展示所有数据，而是先带读者理解“脱钩”的判断逻辑。

项目也考虑过只使用折线图展示 GDP 与 CO2 的时间变化。但折线图需要读者在两条曲线之间来回比较，难以直接看到“经济继续增长、排放开始下降”的转向。因此最终选择“总览散点图 + 连线散点图”的组合：前者建立象限判断，后者解释国家路径如何进入这些象限。

在全球总览中，如果完整展示所有极端值，大多数国家会被压缩在图表一角，无法辨认。直接删除极端值会损害数据完整性，因此最终采用坐标轴截断与边缘标记的折中方案。

国家分型部分也曾考虑用表格列出国家类型，但表格无法展示每一类路径的形状差异。最终采用小多图，让读者通过一组微型趋势图直接比较不同类型的轨迹。为避免分类显得像作者主观判断，每个类型卡片都加入 rule badge，公开 GDP 与人均 CO2 变化的判定逻辑。

- 数据新闻中常见的 small multiples 设计，用于在有限空间中比较多个国家或多种类型的趋势。
- 现代交互式可视化中使用的 focus + context、brush selection、semantic zooming 等技术。

本项目没有直接复用上述作品的代码或视觉素材，只借鉴其叙事与视觉编码思想。所有图表均基于本项目的数据处理管线和 Vue + D3 前端实现。

## 7. 开发过程与分工

项目按以下顺序开发：

- 先确定研究问题与核心叙事：从“谁脱钩了”扩展到“何时脱钩、如何脱钩、是否真实脱钩”，并整理为六步 guided tour。
- 再补齐数据字段：加入 GDP、人均排放、消费端排放、能源结构、碳强度、国家类型等字段。
- 然后拆分页面结构：将页面分为故事、全球图谱、路径机制、类型总结和方法说明。
- 最后打磨交互与部署：修复滚动叙事同步、图表遮挡、悬停高亮、框选选择、GitHub Pages 构建等问题。

因为是个人项目，所有工作由一人完成。大致时间投入约为 50 小时，其中数据清洗与字段设计约 10 小时，核心图表与交互开发约 22 小时，叙事文本与视觉排版约 10 小时，浏览器验收、部署和文档整理约 8 小时。

最耗时的部分是滚动叙事与图表联动。相比静态图表，滚动叙事需要同时处理章节定位、图表状态切换、图表高度、移动端布局和用户手动点击跳转。如果这些细节处理不好，读者会感到文字和图表对不上。另一个耗时点是拥挤散点图的可读性优化，包括坐标轴截断、气泡半径限制、悬停命中区域和选中后的视觉层级。

## 8. 反思与局限

这个项目的主要收获是：碳脱钩不适合只用一个终点指标判断。一个国家是否“成功”，不仅取决于 1990 和 2022 两个年份之间的变化，也取决于中间轨迹是否出现了持续转向，以及这种转向是否建立在真实的能源和产业变化上。

项目仍然存在几个局限：

- 国家尺度会掩盖国内地区、产业和收入群体之间的差异。
- 消费端排放数据比生产端排放更不完整，因此部分国家不能进行同等深度的比较。
- 可视化展示的是相关关系和轨迹变化，不能单独证明某项政策造成了排放下降。
- 坐标轴截断改善了主要国家的可读性，但也意味着极端国家需要通过边缘标记和 tooltip 额外解释。
- 当前机制解释仍偏宏观，如果继续扩展，可以加入电力结构、产业结构、政策节点和能源价格等更细的数据。

## 9. 技术实现与运行方式

前端使用 Vue 3、TypeScript、Vite 和 D3。数据在构建前由 Node.js 脚本预处理成静态 JSON，部署后不需要后端服务。

本地运行方式：

```bash
cd webapp
npm install
npm run dev
```

构建方式：

```bash
cd webapp
npm run build
```

GitHub Pages 部署使用仓库中的 GitHub Actions 工作流。构建产物来自 `dist`，说明文档会随静态资源一起发布。
